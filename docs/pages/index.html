Pages Guide

Front Matter

All files in the pages directory must have the following front matter declared at the beginning of the file.

---
title: "Page Title"
description: "Description."
---

The title property will be used for auto-generated navigation elements and as the page's </code>.</p><p>Both the <code>title</code> and <code>description</code> properties will be used for search functionality.</p></article><article id="soy_files"><h2>Soy</h2><p>All files ending with the <code>.soy</code> extension will be rendered as soy templates.</p><div class="code-container"><button class="btn btn-sm btn-copy"><span class="icon-12-overlap"></span></button><pre><code class="code" data-mode="text/x-soy" ref="code">--- description: "Page description." title: "Page" --- {namespace page} /** */ {template .render} <span>Hello, World!</span> {/template}</code></pre></div><p>Note: all <code>namespace</code> properties must be unique across all files within the <code>src</code> directory, and every soy file must define a <code>.render</code> template.</p><h3>Site/Page Data</h3><p>Every page is passed a <code>site</code> and <code>page</code> param.</p><p>The <code>page</code> param represents the front matter of the current file.</p><div class="code-container"><button class="btn btn-sm btn-copy"><span class="icon-12-overlap"></span></button><pre><code class="code" data-mode="text/x-soy" ref="code">{$page.title} {$page.description}</code></pre></div><p>The <code>site</code> param contains project meta data such as <code>site.title</code> and <code>site.repo</code>. It also contains the structure of the entire site which can be used to generate navigation elements. This structure data is defined as the <code>site.index</code> property representing the index page of the project.</p><div class="code-container"><button class="btn btn-sm btn-copy"><span class="icon-12-overlap"></span></button><pre><code class="code" data-mode="text/x-soy" ref="code">{$site.title} {$site.index}</code></pre></div><p>This data is pulled from the <code>dist/site.json</code> file.
